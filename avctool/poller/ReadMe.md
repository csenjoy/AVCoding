# EventPoller是什么
## 提供接口轮询是否发生I/O事件，通常是网络I/O
    例如使用select或则epoll_wait轮询
## 由于select或epoll_wait通常与时间（唤醒超时有关）因此也尝尝提供定时器功能

# 定时器事件需求分析
## 添加定时器
### 定时器输入
    超时时间和回调函数
### 超时时间
    定时器添加成功时，会根据当前时刻 + 延迟时间得到超时刻度
## 删除定时器
    实际上并不需要删除定时器堆上的定时器节点，只需要使用TaskCancelable接口，完成定时器的取消
## 定时器超时调度
    定时器堆使用multi_map实现
